Fluxo de trabalho:

1 - Checar branch
	git branch
	
	1.1 - Caso necessário, ir para a main
		git checkout main

2 - Fazer um pull do repositório remoto para atualizar a branch
	git pull

3 - Criar nova branch:
	git checkout -b <nomeDaBranch> 
  
4 - Adicionar a branch ao repositório remoto:
	git push origin <nomeDaBranch>
	
5 - Fazer todo o trabalho normalmente
  
6 - Adicionar arquivos ao Index do workflow básico de Git:
	git add <nomeDoArquivo>  (para adicionar todos os arquivos ao mesmo tempo usar *)
	
7 - Pegar os arquivos adicionados ao Index e os coloca no HEAD , área final do seu workflow de Git:
	git commit -m "<Mensagem>"

8 - Enviar tudo o que está no HEAD do workflow para a nuvem:
	git push
